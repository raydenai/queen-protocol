# Decompose Refactor — Meta-Prompt Template (v2.20.0)

> **Role:** Super-Queen auto-decomposer  
> **Input:** A user refactor request (free text, e.g. "rename `getUserData` to `loadUser` across codebase")  
> **Output:** A shard graph in JSON matching §4 schema. Refactors are **single-queen** (read-coherence-bounded).

---

## 1. Input ingestion

Read the refactor description. Classify into one of:

- **Mechanical rename/move:** symbol rename, file move, directory restructure, import path update.
- **Behavior-preserving transformation:** switch from callback → async/await, extract component, inline helper.
- **Type-system refactor:** add strict types, migrate from JS → TS, update interface shapes.
- **Cross-cutting cleanup:** "remove all deprecated v1 API usages" spanning many subsystems.

If the refactor is vague ("clean up the codebase"), refuse and request scope: emit `{"status": "REFUSED", "reason": "scope insufficient", "requested_clarification": [...]}`.

---

## 2. Single-queen rule (§30.3, §29.18)

Refactors are **single-queen by default**, NOT multi-queen. The super-queen assigns the entire refactor to **one queen**, which may internally shard into sub-shards, but those sub-shards are serialized at shared files.

**Why single-queen?** Refactor wins are **read-coherence-bounded**. A rename across 50 files requires a single coherent view of all call sites. Splitting this across queens produces:
- Partial renames (some call sites old, some new)
- Merge conflicts on every shared file
- Type-check failures at converge that are expensive to reconcile

The only exception: a refactor that is **genuinely disjoint by directory** (e.g., "rename all Python test utilities in `tests/` AND rename all React hooks in `hooks/`" with zero shared imports). In that rare case, two parallel queens are permissible, but only after verifying no cross-directory imports exist.

---

## 3. Internal sub-sharding (within the single queen)

The single queen may break the refactor into sub-shards for tracking, but they are **serialized**, not parallel:

```text
Shard A: rename symbol in packages/core/src/**
Shard B: rename symbol in packages/ui/src/**
Shard C: rename symbol in apps/api/src/**
```

All sub-shards share the same `files_allowed` root (e.g., `**/src/**/*`) — they are **sub-shards of one queen**, not independent shards. The queen handles them sequentially in its own SURVEY→PLAN→DISPATCH cycle.

**Do NOT** emit these as separate top-level shards with disjoint `files_allowed` and hope they merge cleanly. They won't.

## 4. Routing tier for refactors

| Condition | Tier | Backend |
|---|---|---|
| Pure rename/codemod, >20 files, pattern is regular | Race (single-queen internal) | `kimi-isolated` (mechanical, cheap, one-shot) |
| Behavior-preserving transform, needs type-check verification | Review | `claude-ant` or `agent:codex-rescue` |
| Cross-cutting cleanup requiring judgment ("remove deprecated v1 usages") | Review | `claude-ant` (reasoning to distinguish safe vs unsafe removals) |
| Type-system refactor with cascading type errors | Race | `claude-ant` + `agent:codex-rescue` tournament if high-stakes |

**Key difference from features:** refactors do NOT use `tournament` for safety the way migrations do. A tournament on a 50-file rename produces N diffs that are all structurally similar but textually incompatible — merge hell. Instead, use `kimi-isolated` for mechanical refactors (fast, deterministic) and `claude-ant` for judgment-shaped refactors.

---

## 5. Lane assignment for refactors

```text
IF complexity == mechanical AND tags & {codemod, rename, format, move}
   → kimi-isolated  (K2.6 is sufficient; deterministic pattern)

IF tags & {async-pr, dep-bump, test-backfill, mechanical-mass-refactor}
   → jules-async  (fire-and-forget PR for bulk mechanical work)

IF behavior-preserving transform requires reasoning about edge cases
   → claude-ant  (Opus handles "is this callback always invoked?")

IF type-system refactor with >100 cascading errors
   → claude-ant  (self-recovery on stuck; iterative fix loops)

DEFAULT
   → claude-ant
```

## 6. Output format

Because refactors are single-queen, the output is a **single shard group** with an explicit `single_queen: true` flag. The super-queen dispatches this to one queen; that queen does its own internal planning.

```json
{
  "schema_version": "2.20.0",
  "colony_id": "<auto-generated: YYYY-MM-DD-refactor-name>",
  "goal": "Refactor: <description>",
  "decomposition_type": "refactor",
  "status": "PLAN",
  "single_queen": true,
  "shards": [
    {
      "id": "s01",
      "title": "Refactor: ...",
      "kind": "code",
      "tags": ["refactor", "rename", "mechanical"],
      "priority": "normal",
      "complexity": "mechanical | complex",
      "files_allowed": ["<broad-glob>/**"],
      "depends_on": [],
      "estimated_lines": 200,
      "skills_required": [],
      "gates": ["typecheck", "ruff check", "pytest <scope>"],
      "deadline_minutes": 30,
      "routing_tier": "solo | review",
      "backend": "kimi-isolated | claude-ant"
    }
  ],
  "dag_validation": {
    "acyclic": true,
    "all_deps_resolve": true,
    "files_overlap": false,
    "has_starting_shard": true
  },
  "assumptions": ["..."]
}
```

**If the refactor spans >100 files**, the super-queen MAY include a `sub_shard_hints` array (advisory only) suggesting directory splits, but the receiving queen decides whether to act on them. The hints are not a plan — they are scaffolding.

## 7. Anti-patterns

1. **Multi-queen refactor** → never. Merge conflicts scale with file count.
2. **Tournament on a rename** → produces N incompatible diffs. Use Kimi for mechanical, Claude for judgment.
3. **Skipping type-check gate** → refactors are behavior-preserving ONLY if the type system agrees. `typecheck` is mandatory.
4. **Dispatching without a pre-refactor snapshot** → the queen must snapshot the tree before starting so rollback is possible if the refactor introduces subtle breakage.
5. **Absorbing a refactor into a feature colony** → a refactor that touches 50 files should not ride along with a feature colony. The feature ants will see moving symbols mid-flight and break. Run refactors as standalone colonies.
