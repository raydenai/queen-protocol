# v2.23.0 — Unified meshboard CLI view: Design Notes

Status: DRAFT ONLY (do NOT commit).

## Goal

Wire the v2.21.2 meshboard-aggregator output into a production-quality,
operator-facing dashboard. Evolves the basic `meshboard-view.sh tail` refresh
loop into a full TUI with color-coded states, sorted queues, ETA estimates,
and blocked-queue dependency visualizations.

## Files

| File | Purpose |
|---|---|
| `meshboard-dashboard.sh` | Full-screen TUI, auto-refresh every 2s |
| `meshboard-summary-line.sh` | One-line summary for SessionStart hooks |
| `design.md` | This document — failure modes, interactions, accessibility |

## Architecture

```
~/.claude/state/super-queen/meshboard.json
              │
              ├──► meshboard-dashboard.sh        (operator TUI)
              │
              └──► meshboard-summary-line.sh     (banner injection)
```

Both are **read-only consumers** of the aggregator output. They do not write
state, lock files, or interact with the queens directly.

## meshboard-dashboard.sh

### Rendering strategy

- **Cursor-home redraw**: emits `\033[H` (move cursor to top-left) followed by
  `\033[J` (clear from cursor to end-of-screen). This is flicker-free on all
  modern terminals because the visible screen buffer is rewritten in-place.
- **No `clear` / `tput clear`**: those commands force a full screen erase and
  cause perceptible flicker, especially on 2s refresh loops.
- **Atomic output**: the entire frame is buffered in a shell variable and
  printed with a single `printf '%b'`, eliminating mid-frame tearing.

### Color mapping (ANSI)

| State | Color | Code |
|---|---|---|
| LAND | Green | `\033[32m` |
| WATCH | Yellow | `\033[33m` |
| BLOCKED | Red | `\033[31m` |
| DEAD | Gray (dim) | `\033[90m` |
| Others | Default | — |

### Sorting

Queens are sorted by:
1. Progress descending (most-complete first — gives operator immediate sense of
   what is about to finish).
2. State severity tie-breaker: BLOCKED → WATCH → CONVERGE → DISPATCH → PLAN →
   LAND → DEAD.
3. Queen ID alphabetical (stable, deterministic).

### ETA estimate

Linear extrapolation from elapsed wall-clock time and completion fraction:

```
elapsed = now - started_at
fraction = done / total
total_estimated = elapsed / fraction
ETA = total_estimated - elapsed
```

- If `done == 0`, ETA is shown as `—` (unknown; no slope yet).
- If state is `LAND`, ETA is `0` (displayed as `—` or implicitly done).
- Rounded to nearest second, formatted as `NNs`, `NNm`, or `NNhNNm`.

This is intentionally simple; it assumes roughly linear progress per shard.
It is accurate for homogeneous shard workloads and becomes a lower-bound
estimate for highly variable shard sizes.

### Blocked-queue visualization

For every queen in state `BLOCKED` with a non-null `blocked_on` field, the
dashboard renders a dependency edge line:

```
→ Queen-C → blocked on Queen-A's API contract
```

If multiple blocked queens exist, each gets its own line in the edges panel.
This makes cross-queen deadlocks visible at a glance without requiring the
operator to scroll through individual rows.

### Footer

Aggregate stats computed across the current `meshboard.json` snapshot:

```
Aggregate: N active  M blocked  K landed  D dead  |  wall-clock Xm
```

- **active**: queens not in `LAND` or `DEAD`.
- **wall-clock**: maximum elapsed time among all non-`LAND` queens (i.e. the
  longest-running in-flight queen).

### Keyboard interactions

| Key | Action |
|---|---|
| `q` / `Q` | Quit cleanly (restore cursor, exit 0) |
| `r` / `R` | Force immediate refresh (skip remaining sleep) |
| `Ctrl+C` | Exit via SIGINT trap — same as `q` |
| Arrow keys, other chars | Ignored (escape sequences drained) |

Input is read via `read -rs -t 0.2 -n 1`, giving a 200ms window per loop
iteration to capture a keystroke before the next sleep. This keeps latency
responsive without burning CPU.

### Auto-refresh

Default interval: **2 seconds**.
Configurable via `REFRESH` environment variable (e.g. `REFRESH=5 ./meshboard-dashboard.sh`).
**Hard floor**: 1 second. If `REFRESH` is set below 1, it is clamped to 1.
This prevents CPU waste and respects the v2.21.2 constraint of not polling
faster than 1s.

## meshboard-summary-line.sh

### Output contract

Exactly one line of text, no trailing newline beyond the shell `echo`.

Examples:

```
[Super-Queen: 3 queens active | Queen-B blocked | ETA 8min]
[Super-Queen: 1 queens active]
[Super-Queen: inactive]
[Super-Queen: 2 queens active | 2 blocked | ETA 1h12min]
```

When multiple blocked queens exist, the summary counts them (`2 blocked`)
rather than listing names, to keep line length bounded.

### ETA in summary line

The summary shows the **maximum ETA among all active queens**. This is the
pessimistic (safe) estimate — the operator knows the session will take at
least this long.

## Failure modes

| Failure | Detection | Behavior |
|---|---|---|
| **meshboard.json missing** | File existence check | Dashboard shows `no super-queen active` in dim gray. Summary line emits `[Super-Queen: inactive]`. Exit code 0 (non-fatal). |
| **meshboard.json corrupted** | `json.load()` exception | Dashboard shows `meshboard.json corrupted` in red. Summary line emits `[Super-Queen: meshboard corrupted]`. |
| **Aggregator stale (>30s)** | Compare `updated_at` in JSON to wall-clock | Dashboard prints `⚠ aggregator stale (NNs ago)` in red above the table. Does NOT auto-exit — operator may intentionally pause the aggregator. |
| **Terminal too narrow** | `tput cols` < 80 | Dashboard falls back to compact layout (one line per queen, fewer columns). No truncation of critical fields. |
| **No color support** | `NO_COLOR` env var or `--plain` flag | All ANSI escapes suppressed. Dashboard renders plain text; summary line unchanged (it has no color). `--plain` mode also disables cursor hide/show and auto-refresh loop — it prints one shot and exits. |
| **Single key read races with refresh** | `read -t 0.2` may miss very fast keypress | Acceptable: next loop iteration catches it. `r` to refresh is idempotent. |
| **SIGTERM / script killed** | `trap` on TERM and INT | Cursor restored, clean exit. No orphaned terminal state. |

## Accessibility

- **NO_COLOR**:遵循 the [no-color.org](https://no-color.org/) convention. If the
  `NO_COLOR` environment variable is set (to any non-empty value), all ANSI
  color and cursor control sequences are suppressed.
- **--plain mode**: Explicit CLI flag `--plain` that disables colors and
  cursor manipulation. In this mode the dashboard prints a single snapshot and
  exits, making it suitable for:
  - Piping to `less`
  - Logging to files
  - CI/CD artifact capture
  - Screen-reader consumption (no continuous redraw)
- **Contrast**: Color choices use standard 16-color ANSI codes. Green/yellow/red
  are distinguishable on both light and dark terminal backgrounds. Gray (`90`)
  is used only for non-critical dim text (timestamps, help line).
- **No animation / no blink**: The only motion is the 2s refresh. No blinking
  cursors, no spinner characters, no rapidly changing elements.

## Performance

- Both scripts are read-only; no locks, no writes.
- Python is invoked once per refresh via a heredoc. On a typical macOS/Linux
  system this costs ~30-50ms — well within the 2s refresh budget.
- For environments where Python startup is slow, a future optimization could
  replace the inline Python with `jq` (if installed) or a small compiled helper.
  The current design prefers Python because `jq` is not guaranteed to be present
  (per project convention in `colony-meshboard-adapter.sh`).

## Future work (out of scope for v2.23.0)

- **Persistent historical bars**: v3.1.x learning phase could render sparkline
  progress bars from a time-series log.
- **Alert on stuck queens**: v2.22.x operational UX will add automatic
  alerting when a queen is BLOCKED for longer than a threshold.
- **Mouse support**: Not planned; keyboard-only keeps the tool simple and
  SSH-friendly.
- **Config file**: If operators need persistent theming or refresh tuning, a
  `~/.claude/config/meshboard` file could be added later.

## Acceptance criteria (for v2.23.0 ship)

- [ ] Dashboard renders correctly with 3+ simulated queens, updates every 2s.
- [ ] Queens sort by progress descending; blocked queens show dependency edges.
- [ ] ETA updates monotonically (or decreases as progress advances) in normal
      linear workloads.
- [ ] `--plain` produces identical information without ANSI codes and exits
      after one frame.
- [ ] Missing or stale `meshboard.json` degrades gracefully with informative
      messages.
- [ ] Summary line fits within 80 columns for typical workloads.
- [ ] Cursor restored and terminal left clean after `q` or Ctrl+C.
